OpenCode Subagent Prompt Injection

The system observes OpenCode runs through Raindrop traces, lets an observer judge decide when a subagent needs steering, resolves the exact OpenCode child session, and injects a prompt into that session through the OpenCode server API.

Plant

OpenCode parent run

Runs the main task and launches task subagents. Must be attached to opencode serve for live REST control.

Sensor

Raindrop Workshop

Stores runs, spans, task outputs, observer debug traces, and steering events in SQLite and streams them in the UI.

Controller

Observer agent

Reads SQLite traces, detects drift or bad work, and emits a corrective decision with a target span and prompt text.

Actuator

Steering actuator

Converts a Raindrop task span into an OpenCode ses_... session and calls prompt_async.

Service Map

OpenCode execution and session control Raindrop trace capture and storage Observer judging and trigger loop Nudge, stop, restart, or system-prompt update Human-facing UI surfaces

Runtime Data Flow

1. OpenCode run Parent session starts and launches a subagent through the task tool.
→
2. Raindrop trace The task span output contains <task id="ses_...">, which is the child session id.
→
3. Observer pass The observer reads spans, outputs, tool calls, and main task context from SQLite.
→
4. Actuator call The observer posts targetSpanId or targetSubagentSpanId plus a nudge.
5. Target resolver The actuator fetches /api/runs/detail/:runId and extracts the child ses_....
→
6. OpenCode REST It calls POST /session/:id/prompt_async with noReply: true.
→
7. UI event The actuator writes an applied or failed steering event back to Workshop.
→
8. Human view Observer tab shows only big actions. Observer Debug shows compact inputs, outputs, and tool calls.

What Was Built

1

Subagent target resolution

targetSpanId or targetSubagentSpanId can be a Raindrop task span. The actuator extracts the child OpenCode session from that span output.

2

Direct session targeting

If the observer already has sessionId: "ses_...", the actuator skips trace lookup and targets that OpenCode session directly.

3

Parent fallback

If no child session can be found, the actuator falls back to the parent run.convo_id so it can steer the coordinator.

4

Dry-run verification

POST /resolve proves how a target resolves without calling OpenCode or writing a Workshop event.

API Shape

POST http://localhost:3032/apply { "observedRunId": "d339...", "targetSpanId": "4d7526759abf5c41", "action": "nudge", "afterPrompt": "Observer nudge: refocus..." }
POST http://localhost:4096/session/ses_.../prompt_async { "noReply": true, "parts": [ { "type": "text", "text": "Observer nudge: refocus..." } ] }

Important runtime requirement

Live prompt injection requires OpenCode to be reachable through opencode serve. A standalone opencode run process still emits Raindrop traces, but it is not controllable over REST unless it is run through or attached to the server.

The intended launch shape is opencode serve --port 4096 and then opencode run --attach http://localhost:4096 ....
