Raindrop Live Observer Agent

Build a live Workshop experience where a separate local OpenCode observer agent is visible in Raindrop, evaluates subagents in real time, emits its own trace, and records nudges or prompt updates back onto the original subagent timeline.

The Product Idea

Raindrop should not only show what the main agent and subagents did. It should also show the observer agent that is watching them, explain what it noticed, and display the steering action it took or proposed.

Observer Is First-Class

The observer agent emits normal Raindrop traces with event name observer_agent_session. Those traces are linked to the observed OpenCode run through metadata like observedRunId.

Steering Is Durable

Nudges, stop/restart recommendations, and prompt updates are stored as steering_events. They appear live in the UI and remain visible when the run is reopened.

System Flow

1. Main OpenCode Agent

Runs locally and emits the normal Raindrop trace. Subagents appear as task/tool spans with child LLM and tool spans.

→

2. Workshop

Stores runs, spans, live events, and steering events in SQLite. Broadcasts updates over the existing WebSocket.

→

3. Observer OpenCode Agent

Runs on a different localhost port, reads Workshop SQLite or APIs, emits its own trace, then posts steering actions back to Workshop.

What The Observer Watches

  • ✓Repeated glob calls returning no files.
  • ✓Failed read calls, especially wrong paths.
  • ✓Subagents running longer than expected.
  • ✓Repeated tool usage without new information.
  • ✓Subagent output that contradicts trace evidence.

What The Observer Writes Back

  • 1Nudge: tell a subagent to change strategy.
  • 2Prompt update: record before/after prompt text.
  • 3Stop: recommend or trigger stopping a subagent.
  • 4Restart: recommend or trigger restart with revised context.
  • 5Note: explain a finding without steering.

Backend Build Plan

  • 1Add steering_events table and migration.
  • 2Add create/list/update helpers in the Workshop DB layer.
  • 3Add POST /api/steering/events for the observer agent.
  • 4Add GET /api/runs/:id/steering for the UI.
  • 5Broadcast steering events over the existing WebSocket.
  • 6Find observer runs by metadata containing observedRunId.

Frontend Build Plan

  • 1Add an Observer tab or lane in run detail.

Observer Trace Metadata

The observer agent should emit a normal Raindrop trace so its own reasoning is visible in Workshop. 

{
  "event": "observer_agent_session",
  "input": {
    "observedRunId": "main-opencode-run-id",
    "snapshot": "subagent summary and recent tool failures"
  },
  "properties": {
    "role": "observer",
    "observedRunId": "main-opencode-run-id",
    "observedEventName": "opencode_session"
  },
  "output": {
    "decision": "nudge",
    "targetSubagentSpanId": "subagent-task-span-id",
    "proposedPrompt": "First verify the repo root...",
    "reason": "Repeated empty glob calls plus failed read."
  }
}
