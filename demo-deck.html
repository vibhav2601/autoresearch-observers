The problem

Agent swarms drift, loop, and contradict each other.

When a fleet of agents fans out on a hard question, the same coordination failures show up run after run. The traces capture all of it, but they get read after the fact instead of during the run.

What's missing

The orchestrator dispatches workers and waits. Nothing reads what they're doing while they run, so problems only surface in the dashboard after the run is over.

Traces aren't being used

Tool calls, messages, and spans already stream to a local store. That stream could be a runtime signal, but today it's just a log to grep later.

Four common failure patterns

1
Duplicate work. Two searchers chase the same query against the same source.
2
Contradiction. Sibling agents return opposing claims and neither notices.
3
Stall. A subagent loops on a tool call; downstream synthesis blocks.
4
Drift. A worker wanders off the assigned subquestion into adjacent territory.
The solution

Live trace monitoring & correction.

An external observer reads spans as they arrive, looks for one of the four patterns, and injects a corrective prompt into the offending subagent — or aborts it. The trace becomes the sensor for a control loop that runs alongside the workers.

Hybrid policy

Cheap heuristics on span features — repeat count, novelty score, coverage overlap, duration — decide when the observer wakes. The LLM decides what to do, returning a typed Intervention.

Rate-limited nudges

Confidence-gated and throttled, since a thrashing controller is worse than none. Low-confidence calls land as proposed in the UI and never get injected.

Audit on the same timeline

Each diagnosis and nudge is written back as its own span on the run it corrected. That's the demo overlay, and the input for replay-based learning later.

Benchmark

Same prompt, observer off vs on.

An OBSERVER_DISABLED=1 env var turns the observer's tick loop into a no-op. The bench spawns opencode run --format json, sums tokens and cost from step_finish events, and times spawn-to-exit. Run it twice and flip the toggle in between.

Test scenario: hallucinating subagents

The fixture launches two intentionally disagreeing OpenCode subagents on a small repo. Off: they expand parallel, contradictory evidence paths and the parent folds in the wrong answer. On: the observer notices the divergence and injects a refocus prompt into the bad-evidence hunter, which shows up as a steering span on the same trace.

One run per condition is a wiring spot-check, not a measurement. LLM nondeterminism dominates at n=1; results.jsonl is there so we can jq over many runs once the loop is stable.
Architecture

Five local processes.

Everything runs on the laptop. The Raindrop plugin emits spans into Workshop without a fork, the observer is a separate OpenCode process, and the actuator is a small sidecar that maps run and span IDs to live sessions.
